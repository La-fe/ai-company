# Competitor Teardown

竞品深度拆解插件 — 从公开信息逆向工程竞品的产品架构。

## 核心能力

- **逐竞品六层分析**: 问题空间 → 数据模型(分类+关系+字段) → 用户旅程(流程图) → 定价映射 → 留存机制 → Feature目录+设计决策
- **跨竞品模式提炼**: 实体共性矩阵、架构模式、定价对比、留存机制、空白机会
- **多来源信息合并**: 截图素材 → 已有档案 → Web 采集

## 快速开始

```bash
/teardown whop.com stan.store durable.co --output ~/analysis/
```

## 输出

- `competitors/{name}.md` — 每竞品深度档案（六层结构）
- `landscape.md` — 跨竞品模式提炼报告

## 参数

| 参数 | 说明 |
|------|------|
| `$1..N` | 竞品 URL 或名称 |
| `--materials PATH` | 截图/笔记素材目录 |
| `--existing PATH...` | 已有竞品档案 |
| `--output PATH` | 输出目录 |
| `--skip-ai` | 跳过 AI 能力审计 |

## 分析深度

每个竞品 6 层分析：

1. **问题空间** — 具体用户画像、JTBD、触发场景、替代方案、关键指标
2. **数据模型** — 实体四类分类 + 标准化关系图 + 3-5核心实体字段定义
3. **用户旅程** — 旅程表(步骤×实体×CRUD) + ASCII流程图(分叉/循环) + TTFV
4. **定价映射** — 定价模型分类 + 实体↔定价边界表 + 交易费用结构
5. **留存机制** — 7种留存策略×涉及实体×强度 + 迁移成本评估
6. **Feature目录+设计决策** — 完整Feature清单 + AI审计(可选) + 设计决策推理表

完成后自动进入 Phase 5 跨竞品综合。

## 与 cpo-analyze 的区别

| 维度 | `/teardown` | `/cpo-analyze` |
|------|-------------|---------------|
| 深度 | 深而窄 | 浅而广 |
| 核心 | 数据模型 + 实体关系 | 功能列表 + 定价 + 渠道 |
| 每竞品产出 | ~500 行结构化档案 | ~30 行摘要 |
| 适用 | 产品架构决策前 | 快速市场扫描 |
