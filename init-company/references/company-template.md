# Company Template

生成 `company.md` 时按以下结构填充。每个 section 用 H2 分隔，字段用列表或表格。信息缺失时标注 `[TODO: 描述]`。

---

## 结构定义

```markdown
# {公司名称}

> {一句话定位：帮 [谁] 解决 [什么问题]，通过 [什么方式]}

## Identity（身份）

- 领域:
- 阶段: 种子期 / 早期 / 成长期 / 成熟期
- 创立: YYYY
- 语言: 中文 / English / ...
- 官网: [URL]

## Product（产品）

### {产品名称}
- 描述: 一句话说清楚这个产品做什么
- 核心功能:
  1. xxx
  2. xxx
  3. xxx
- 状态: active / beta / coming_soon / discontinued
- 定价: [定价模式 + 具体价格]
- 差异化功能: [和竞品相比独有的]

### 经济引擎
- 获客成本 (CAC): [如有]
- 客单价 (ARPU): [如有]
- 毛利率: [如有]
- 回本周期: [如有]

## Market（市场）

### ICP（理想客户画像）
- 谁: [具体描述，不要泛人群]
- 行业/规模: [细分]
- 付费意愿: [证据或判断]

### JTBD（核心任务）
当 [场景] 时，我想要 [任务]，以便 [期望结果]。

### 用户画像

#### Persona 1: {名称}
- 场景:
- 核心需求:
- 痛点:
- 期望收益:
- 渠道偏好:
- 内容偏好:

#### Persona 2: {名称}
...

### 市场信号
- 留存/复购: [数据或判断]
- NPS/口碑: [数据或判断]
- 市场规模: [TAM/SAM/SOM]

## Position（定位）

### 价值主张
帮 [ICP] 解决 [核心问题]，通过 [机制]，实现 [可衡量结果]。

### 差异化机制
不是形容词，是机制：
1. [数据优势 / 流程嵌入 / 网络效应 / 成本结构 / ...]
2. ...

### 护城河
- 当前: [已有的防御]
- 构建中: [正在积累的]

### 反定位（不做什么）
- 不做 xxx（留给 yyy）
- 不服务 xxx（他们需要 zzz）

## Landscape（战场）

### 竞品矩阵

| 竞品 | 核心定位 | 强项 | 弱项 | 价格 | 流量策略 |
|------|---------|------|------|------|---------|
| A    |         |      |      |      |         |
| B    |         |      |      |      |         |

### 替代方案
- 用户现在怎么解决这个问题？（人工? 通用工具? 不解决?）

### 市场力量
- 供给侧趋势:
- 需求侧趋势:
- 技术变化:

## Principles（原则）

### 决策准则
1. xxx > yyy
2. ...

### 红线
- 不做: xxx
- 不说: xxx

### 沟通风格
- 调性:
- 禁忌:
```

---

## 质量评估标准

| Section | 合格标准 | 优秀标准 |
|---------|---------|---------|
| Identity | 30 秒能回答"做什么的" | 外行也能理解 |
| Product | 能回答"卖什么、多少钱" | 有经济引擎数据 |
| Market | ICP 具体到一类人 | 有数据证据（留存、付费） |
| Position | 差异化是机制不是形容词 | 有反定位（不做什么） |
| Landscape | 列出 2+ 竞品 | 有替代方案分析 |
| Principles | 有 2+ 决策准则 | 准则能直接指导模棱两可的决策 |
