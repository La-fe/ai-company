# CMO Growth Plan RLHF — 增长策略质量进化循环

> 每一轮增长策略都是一次学习机会。通过结构化评估 → 反馈 → 进化，让增长计划的质量和 ROI 持续递增。

---

## 核心理念

增长策略不是一次性输出，而是持续进化的闭环。RLHF 循环将每次增长计划的执行结果转化为可量化的信号，驱动渠道选择、预算分配、执行策略的迭代升级。

核心假设：
- 增长策略质量可以通过多维度评分客观衡量
- 用户反馈 + 实际 ROI 数据是最有价值的奖励信号
- 渠道策略的有效性可以通过累积执行数据验证
- 低 ROI 的渠道组合应该被自动弱化，高 ROI 的应该被强化

---

## 5 阶段循环

```
┌──────────────────┐
│     Phase 1      │
│  Generate        │──→ CMO 按流程生成增长计划
│  (生成增长计划)   │
└────────┬─────────┘
         ▼
┌──────────────────┐
│     Phase 2      │
│  Evaluate        │──→ 多维度评估打分
│  (策略质量评估)   │
└────────┬─────────┘
         ▼
┌──────────────────┐
│     Phase 3      │
│  Feedback        │──→ ⏸️ 等待用户反馈 + ROI 数据
│  (反馈收集)      │
└────────┬─────────┘
         ▼
┌──────────────────┐
│     Phase 4      │
│  Update          │──→ 更新策略规则置信度
│  (策略更新)      │
└────────┬─────────┘
         ▼
┌──────────────────┐
│     Phase 5      │
│  Consolidate     │──→ 固化经验，淘汰低效策略
│  (整合优化)      │
└────────┬─────────┘
         │
         └──→ 回到 Phase 1（下一轮增长计划）
```

### Phase 1: Generate — 生成增长计划

CMO 按照标准流程生成增长计划产物。

**执行要求**：
- 按 cmo-plan 标准流程生成 growth-plan.md
- 记录渠道选择决策点和所依赖的假设
- 标注哪些策略规则被应用了（用于后续追踪规则有效性）
- 记录渠道评分矩阵的打分依据

**记录格式**：

```markdown
## 策略决策日志

### 应用的规则
- rule_id: growth-001 — "冷启动阶段优先低成本+快见效渠道"
- rule_id: growth-005 — "预算 <10k 不同时启动超过 3 个渠道"

### 渠道选择假设
1. 假设: ICP 活跃在 {平台} — 验证方式: 前 30 天互动数据
2. 假设: SEO 可在 3 个月内见效 — 验证方式: 关键词排名追踪

### 预算分配依据
- 选择 70/20/10 分配而非平均分配，因为 [原因]
- SEM 预算设为 X 而非 Y，因为 [行业 CPC 基准数据]
```

### Phase 2: Evaluate — 多维度评估

对生成的增长计划进行结构化评估。

**执行流程**：
1. 读取评估维度定义: `growth-eval-dimensions.md`
2. 确定当前公司阶段（冷启动 / 增长期 / 成熟期），应用动态权重调整
3. 对增长计划逐维度打分（1-10 分）
4. 检查否决规则（Veto Rules）
5. 生成评估报告

**评估约束**：
- 打分必须给出具体理由，不允许笼统评价
- 每个维度至少列出 1 个扣分项或加分项
- 否决规则优先于总分判断
- 渠道选择必须有数据支撑（行业基准、竞品情报）

### Phase 3: Feedback — 人工反馈 + ROI 数据

**⏸️ WAIT — 必须暂停等待用户输入。**

**展示内容**：
1. 评估报告摘要（总分 + 各维度分数）
2. 关键风险点（评分最低的 2 个维度）
3. 改善建议（Top 3）
4. 预估 ROI vs 实际 ROI 对比（如有历史数据）

**收集内容**：

**显式反馈**：
- 用户对每个维度评分的同意/调整
- 用户修改意见及原因（这是最重要的奖励信号）
- 用户对增长计划的整体判断: 执行 / 调整 / 重做

**数据反馈（30/60/90 天检查点可用时）**：
- 各渠道实际 ROI 数据
- 各渠道实际 CPA/CAC 数据
- 关键 KPI 达成率（vs 预估目标）
- 预算消耗 vs 计划对比

**反馈记录格式**：

```markdown
## 用户反馈

### 维度评分调整
| 维度 | AI 评分 | 用户评分 | 差异 | 用户说明 |
|------|--------|---------|------|---------|
| 渠道选择合理性 | 7 | 8 | +1 | "SEO+社媒组合确实适合当前阶段" |
| ROI 预估准确度 | 8 | 5 | -3 | "SEM 的 CPC 估低了，实际高 2 倍" |

### 整体判断
- 决定: [执行 / 调整 / 重做]
- 理由: ...

### ROI 数据反馈（如可用）
| 渠道 | 预估 ROI | 实际 ROI | 差异 | 原因分析 |
|------|---------|---------|------|---------|
| SEO | 5:1 | 3:1 | -40% | 关键词竞争比预估激烈 |
| SEM | 3:1 | 4:1 | +33% | 品牌词 CPC 低于预期 |
```

### Phase 4: Update — 策略规则更新

基于反馈信号更新增长策略规则的置信度。

**更新公式**：

```
Score = Σ(applied_count × feedback_signal) / total_applications
Confidence = 1 - 1/(applied_count + 1)
```

其中:
- `applied_count`: 该规则被应用的次数
- `feedback_signal`: +1（正面反馈/ROI 达标）/ 0（中性）/ -1（负面反馈/ROI 未达标）
- `total_applications`: 所有规则的总应用次数

**ROI 信号映射**：

```
实际 ROI >= 预估 ROI × 1.2  → feedback_signal = +1.0（超预期）
实际 ROI >= 预估 ROI × 0.8  → feedback_signal = +0.5（基本达标）
实际 ROI >= 预估 ROI × 0.5  → feedback_signal = 0.0（需优化）
实际 ROI < 预估 ROI × 0.5   → feedback_signal = -0.5（需重大调整）
实际 ROI < 预估 ROI × 0.3   → feedback_signal = -1.0（策略失败）
```

**突变策略**：

| 条件 | 动作 | 说明 |
|------|------|------|
| score > 0.7 且 confidence > 0.7 | 强化规则 | 提升权重，标记为"已验证" |
| 0.3 < score < 0.7 | 保持观察 | 不调整，继续收集数据 |
| score < 0.3 且 confidence > 0.5 | 弱化或废弃 | 降低权重，3 次连续低分则废弃 |
| confidence < 0.3 | 探索性调整 | 数据不足，保留但标记"待验证" |

**规则状态流转**：

```
candidate → active → verified → core_rule
                   ↘ deprecated → removed
```

### Phase 5: Consolidate — 固化经验

将本轮学习成果固化为持久化知识。

**固化规则**：
- 高分规则（score > 0.7 且 confidence > 0.7）→ 固化为渠道 SOP 级别规则
- 低分规则（score < 0.3 且 confidence > 0.5）→ 标记为 deprecated，准备废弃
- 新发现的渠道策略模式 → 生成新的 candidate 规则
- 反复出现的 ROI 偏差模式 → 更新渠道 profile 中的 KPI 基准值

**输出产物**：
- 更新 `evolution-state.yaml`（如存在）
- 更新相关渠道 profile（channels/*.md）中的基准数据
- 记录本轮进化日志
- 如有 SOP 级别规则变更，更新相关 reference 文件

**进化日志格式**：

```markdown
## 进化日志 — {date}

### 规则变更
| rule_id | 变更类型 | 变更前 | 变更后 | 原因 |
|---------|---------|--------|--------|------|
| growth-001 | score_update | 0.55 | 0.72 | SEO 策略连续 3 个月超预期 |
| growth-008 | deprecated | active | deprecated | SEM 品类词策略 ROI 持续低于 1:1 |
| growth-NEW-01 | created | — | candidate | 用户反馈小红书种草→转化链路有效 |

### 渠道基准更新
| 渠道 | KPI | 旧基准 | 新基准 | 数据来源 |
|------|-----|--------|--------|---------|
| SEO | 月有机流量增长 | 15% | 12% | 过去 3 个月实际数据 |
| SEM | B2B CPC | ¥15 | ¥22 | Google Ads 实际数据 |

### 累积统计
- 总规则数: X
- 活跃规则: X
- 已验证规则: X
- 废弃规则: X
- 本轮新增: X
- 本轮废弃: X
```

## 评估触发时机

| 时机 | 评估类型 | 说明 | 是否必须 |
|------|---------|------|---------|
| growth-plan.md 生成后 | 策略质量评估 | 评估渠道选择、预算分配、执行计划 | 必须 |
| 30 天检查点 | 早期数据评估 | 对比前 30 天实际数据 vs 计划 | 必须 |
| 60 天检查点 | 中期效果评估 | 渠道 ROI 排序，预算重分配建议 | 必须 |
| 90 天检查点 | 季度复盘评估 | 全面复盘，生成下一周期策略 | 必须 |
| ROI 数据可用时 | 数据驱动评估 | 用实际 ROI 修正策略规则 | 强烈建议 |
| 用户主动触发 | 按需评估 | 渠道表现异常或市场环境变化时 | 任何时候 |
