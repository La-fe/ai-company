# Landing Page — CPO 评审清单

> 类型: landing-page
> 适用: 单页面着陆页，用于广告投放、活动推广、产品发布等需要高转化率的场景。与完整 Website 的区别：Landing Page 只有一个目标、一个页面、一个 CTA。
> 版本: v1.0

---

## 核心考量维度

### 用户路径

Landing Page 的用户路径极短且高度聚焦——从进入到行动，越短越好：

```
Ad/Search/Social → Landing Page → 理解价值 → 信任建立 → CTA 行动
```

**关键路径说明：**

| 触点 | 用户行为 | 用户心理 | 设计要点 |
|------|---------|---------|---------|
| **进入** | 从广告/搜索/社交点击进入 | "这是我搜的/看到的那个东西吗？" | 消息匹配（Message Match）：Landing Page 的标题必须和广告文案一致 |
| **首屏理解** | 扫描 Hero Section | "这是做什么的？对我有什么用？" | 5 秒规则：用户在 5 秒内能说出这个页面是干什么的 |
| **信任建设** | 浏览社会证明、案例 | "靠谱吗？别人用得怎样？" | 具体数据 > 模糊好评；真实客户 > 虚构形象 |
| **CTA 行动** | 点击按钮/填写表单 | "值得一试吗？风险大吗？" | 降低承诺感：免费试用 > 立即购买；减少表单字段 |

**分支路径：**
- **高意向路径**：进入 → 直接点 CTA（不看其他内容）——不要阻挡这类用户
- **犹豫路径**：进入 → 向下滚动 → 看案例/FAQ → 回到顶部 → 点 CTA
- **离开路径**：进入 → 觉得不相关 → 离开——靠消息匹配和精准投放减少此路径

### 关键转化节点

| 转化节点 | 目标行为 | 行业基准 CVR | 关键影响因素 |
|---------|---------|-------------|-------------|
| **首屏 CTA** | 点击首屏的主 CTA 按钮 | 5-15%（暖流量） | 价值主张清晰度、CTA 按钮对比度、消息匹配度 |
| **社会证明区** | 看完社会证明后增强信心 | — (辅助转化) | 客户 Logo 数量、推荐语具体程度、数据可信度 |
| **定价/Offer 区** | 理解报价并决定行动 | — (辅助转化) | 价值感知 > 价格感知、对比锚定、限时紧迫感 |
| **底部 CTA** | 犹豫用户的最终转化 | 3-8% | 重申核心承诺、消除最后顾虑、低承诺 CTA 文案 |

**转化漏斗优化优先级：**
1. **消息匹配** — 如果广告说的和 Landing Page 说的不一样，用户秒退
2. **首屏 CTA** — 首屏决定了大部分用户是否继续
3. **表单优化** — 每多一个字段，转化率下降 10-15%

### 必备功能清单

#### P0 — 没有不行

| # | 功能 | 解决什么问题 | 验收标准 |
|---|------|------------|---------|
| 1 | **Hero Section + 价值主张** | 5 秒内告诉用户"这是什么、对你有什么用" | 一句话主标题 + 一段话副标题 + 一个视觉元素（图/视频），非目标用户也能理解 |
| 2 | **Benefits（不是 Features）** | 用户关心"我能得到什么"而不是"你有什么技术" | 3-5 个核心收益，每个用用户语言描述，有配图或 icon |
| 3 | **社会证明** | 降低信任门槛 | 至少包含以下之一：客户 Logo 墙 / 用户推荐语 / 关键数据（"已服务 X 客户"） |
| 4 | **单一清晰 CTA** | 避免选择瘫痪 | 全页只有一种 CTA 行动（可多个入口但同一目标），按钮文案是动词（"开始免费试用"而非"提交"） |
| 5 | **Mobile-first 设计** | Landing Page 广告流量 60%+ 来自手机 | 在 375px 宽度下所有内容可读可交互，CTA 按钮可单手触达 |

#### P1 — 强烈建议

| # | 功能 | 价值说明 | 建议加入版本 |
|---|------|---------|-------------|
| 1 | **FAQ 区** | 消除常见疑虑，减少跳出 | V1.0 即可加 |
| 2 | **A/B 测试能力** | Landing Page 的核心优化手段 | V1.0 必须有，哪怕用最简单的方案 |
| 3 | **退出意图弹窗** | 挽回即将离开的用户 | V1.1 |
| 4 | **UTM 参数追踪** | 区分不同流量来源的效果 | V1.0 埋点时一起做 |
| 5 | **感谢页/确认页** | 转化后引导下一步行动 | V1.0 |

### 常见陷阱

| # | 陷阱 | 典型表现 | 后果 | 预防措施 |
|---|------|---------|------|---------|
| 1 | **多个 CTA** | 页面上有"免费试用""预约演示""下载白皮书"三个不同按钮 | 用户不知道该点哪个，转化率被稀释 | 一个页面一个目标，想要多个 CTA 就做多个 Landing Page |
| 2 | **Feature Dumping** | 列了 20 个功能特性 | 用户信息过载，什么也没记住 | 最多 5 个核心收益，用用户语言而非技术语言 |
| 3 | **无社会证明** | 没有任何第三方背书 | 用户没有信任基础，不敢行动 | 至少有一种社会证明（Logo / 推荐语 / 数据） |
| 4 | **加载速度慢** | 大图片 + 重 JS + 无 CDN | Landing Page > 3 秒加载，广告费打水漂 | 图片压缩、懒加载、关键路径 CSS 内联 |
| 5 | **表单要太多信息** | 注册要填公司名、职位、电话、地址 | 每多一个字段丢失 10-15% 的潜在转化 | MVP 只要邮箱；用 progressive profiling 后续补充信息 |
| 6 | **消息不匹配** | 广告说"免费 AI 工具"，Landing Page 标题是"企业级解决方案" | 用户觉得被骗，秒退 | Landing Page 标题必须和广告文案核心信息一致 |
| 7 | **导航栏分散注意力** | 放了完整的网站导航栏 | 用户点导航去了其他页面，转化丢失 | Landing Page 去掉导航栏，或只保留 Logo |

### 技术考量

#### 架构选型

| 决策点 | 选项 A | 选项 B | 推荐 | 考量因素 |
|--------|--------|--------|------|---------|
| **建站方式** | 代码定制（Next.js/Astro） | 无代码工具（Unbounce/Carrd） | 视团队而定 | 高频迭代 + 非技术团队 → 无代码；需要深度定制 → 代码 |
| **A/B 测试** | 内置代码层方案（Vercel Edge Config） | 第三方工具（Google Optimize 替代品 / VWO） | 第三方工具 | 快速启动、非开发人员可操作 |
| **表单处理** | 自建后端 | 第三方服务（Typeform/Formspree） | 第三方 → MVP | 快速上线 > 完全控制；数据量大后再迁移 |
| **托管方式** | 静态部署（Vercel/Netlify） | 动态服务器 | 静态部署 | Landing Page 几乎没有动态需求，静态 = 快 + 便宜 |

#### 性能要求

| 指标 | 目标值 | 测量方式 |
|------|--------|---------|
| 首屏加载时间 | < 1.5s（WiFi），< 3s（4G） | Lighthouse + WebPageTest |
| LCP | < 2.0s | Web Vitals |
| 总页面大小 | < 1MB（gzipped） | Chrome DevTools Network |
| CLS | < 0.05 | Web Vitals（Landing Page 对视觉稳定性要求更高） |
| 图片优化 | WebP/AVIF 格式，懒加载非首屏图 | Lighthouse |

#### A/B 测试基础设施

Landing Page 的核心优化手段就是 A/B 测试，技术层必须支持：

| 测试维度 | 说明 | 优先级 |
|---------|------|--------|
| 标题文案 | 不同的价值主张表述 | 最高 |
| CTA 按钮 | 文案、颜色、位置 | 高 |
| Hero 图/视频 | 视觉元素对转化的影响 | 中 |
| 社会证明 | 不同类型社会证明的效果 | 中 |
| 表单长度 | 字段数量对转化率的影响 | 高 |

#### Heatmap + 行为追踪

| 工具 | 用途 | 建议 |
|------|------|------|
| Hotjar / Microsoft Clarity | 点击热力图 + 滚动深度 | 上线第一天就接入 |
| Session Recording | 回放用户实际行为 | 每周看 10-20 个录像 |
| Form Analytics | 表单每个字段的放弃率 | 优化表单的核心数据 |

### 行业基准

| KPI | 行业基准 | 优秀水平 | 合格水平 | 数据来源 |
|-----|---------|---------|---------|---------|
| **转化率（暖流量）** | 5-15% | > 20% | > 5% | Unbounce Conversion Benchmark Report |
| **转化率（冷流量）** | 2-5% | > 8% | > 2% | Unbounce Conversion Benchmark Report |
| **跳出率** | 60-80% | < 50% | < 70% | 行业平均 |
| **平均页面停留** | 45s - 2min | > 2min | > 1min | Google Analytics |
| **滚动深度（到底部）** | 20-40% | > 50% | > 25% | Hotjar 行业数据 |
| **CTA 点击率** | 3-10% | > 15% | > 5% | 综合行业数据 |
| **表单完成率** | 40-60% | > 70% | > 40% | Formstack |

**基准使用注意：**
- "暖流量"指已有品牌认知或意向的用户（如 retargeting 广告、品牌搜索）
- "冷流量"指首次接触的用户（如 Facebook 兴趣投放、通用关键词搜索）
- 不同行业差异巨大：SaaS 的 CVR 通常低于电商，但客单价高
- B2B Landing Page 的表单字段通常多于 B2C，CVR 基准也不同
- 首次上线的 Landing Page 转化率通常在基准的 50-70%，A/B 测试 3-5 轮后接近基准
