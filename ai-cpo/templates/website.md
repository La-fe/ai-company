# Website 产品 — CPO 评审清单

> 类型: website
> 适用: 完整的网站产品（非单页 Landing Page），包括 SaaS 官网、电商独立站、内容平台等需要多页面、多功能的网站
> 版本: v1.0

---

## 核心考量维度

### 用户路径

典型的网站用户路径从外部流量入口到最终转化形成完整闭环：

```
Landing → Explore（浏览了解）→ Demo/Trial（体验产品）→ Register（注册）→ Payment（付费）→ Onboarding（激活使用）
```

**关键路径说明：**

| 触点 | 用户行为 | 用户心理 | 设计要点 |
|------|---------|---------|---------|
| **Landing** | 首次进入网站 | "这是什么？跟我有关吗？" | 3 秒内传递核心价值主张 |
| **Explore** | 浏览产品功能、案例、价格 | "这能解决我的问题吗？值不值得？" | 信息架构清晰，降低认知负荷 |
| **Demo/Trial** | 试用或观看演示 | "让我试试看效果怎么样" | 零门槛体验，快速看到价值 |
| **Register** | 创建账号 | "我需要给出个人信息" | 最少必填字段，支持第三方登录 |
| **Payment** | 选择套餐、完成支付 | "这个价格合理吗？安全吗？" | 透明定价，安全感建设 |
| **Onboarding** | 开始使用核心功能 | "怎么开始？我该先做什么？" | 引导流程清晰，快速到达 Aha Moment |

**分支路径：**
- **内容导入路径**：Blog/SEO → 文章 → 兴趣激发 → 产品页 → 转化（适合内容营销驱动的产品）
- **社交导入路径**：社交媒体 → 推荐/分享页 → 注册 → 使用（适合有社交裂变的产品）
- **直接搜索路径**：品牌搜索 → 首页 → 快速注册/登录（老用户或已有认知的用户）

### 关键转化节点

| 转化节点 | 目标行为 | 行业基准 CVR | 关键影响因素 |
|---------|---------|-------------|-------------|
| **Landing Page CTA** | 点击"了解更多"或"免费试用" | 3-7% | 价值主张清晰度、CTA 可见性、页面加载速度 |
| **注册表单** | 完成注册流程 | 20-40% (从表单页) | 字段数量（≤4 个）、第三方登录支持、隐私安全感 |
| **定价页** | 选择套餐 | 10-25% (从定价页) | 套餐对比清晰度、免费试用可用性、社会证明 |
| **Checkout** | 完成支付 | 30-60% (从 checkout) | 支付方式多样性、安全标识、流程步骤数 |
| **Onboarding** | 完成激活（到达 Aha Moment） | 25-50% | 引导流程长度、第一次价值体验时间、帮助文档质量 |

**转化漏斗优化优先级：**
1. **Onboarding 激活** — 注册后不激活等于白注册，激活是留存的前提
2. **Landing Page CTA** — 入口转化率直接决定整个漏斗的效率
3. **注册表单** — 减少摩擦，能不问就不问

### 必备功能清单

#### P0 — 没有不行

| # | 功能 | 解决什么问题 | 验收标准 |
|---|------|------------|---------|
| 1 | **响应式设计** | 50%+ 流量来自移动端 | 在 iPhone SE ~ iPad Pro 所有尺寸下正常显示，可交互 |
| 2 | **快速加载** | 慢速加载直接杀死转化率 | 首屏加载时间 < 3 秒（3G 网络），LCP < 2.5s |
| 3 | **清晰的价值主张** | 用户 3 秒内判断去留 | Hero Section 一句话说清"为谁解决什么问题"，非技术人员能理解 |
| 4 | **SEO 基础** | 自然搜索是长期免费流量 | 页面有 meta title/description，URL 结构清晰，有 sitemap.xml |
| 5 | **数据追踪** | 没有数据就没有优化依据 | 核心转化事件（注册、付费、激活）有埋点，至少接入 Google Analytics |

#### P1 — 强烈建议

| # | 功能 | 价值说明 | 建议加入版本 |
|---|------|---------|-------------|
| 1 | **A/B 测试基础设施** | 持续优化转化率的前提 | V1.1 |
| 2 | **用户行为分析** | Heatmap + Session Recording | V1.1 |
| 3 | **国际化（i18n）** | 如果目标市场涉及多语言 | V1.1 |
| 4 | **无障碍访问（a11y）** | 覆盖更多用户 + 法律合规 | V1.2 |
| 5 | **内容管理系统** | Blog/文档的持续更新能力 | V1.1 |
| 6 | **社交分享** | 用户自发传播 | V1.2 |

### 常见陷阱

| # | 陷阱 | 典型表现 | 后果 | 预防措施 |
|---|------|---------|------|---------|
| 1 | **上线前过度建设** | MVP 做了 20 个页面 | 3 个月没上线，假设没被验证 | MVP 只需要 Landing + 核心功能 + 注册，其他后面加 |
| 2 | **忽略移动端** | "先做桌面端再适配" | 超过一半的用户看到的是破碎的页面 | Mobile-first 设计，始终在手机上预览 |
| 3 | **没有 clear CTA** | 页面上有 5 个按钮，不知道该点哪个 | 用户选择瘫痪，转化率暴跌 | 每个页面一个主 CTA，其他都是次级 |
| 4 | **页面太多** | MVP 就规划了 15 个页面 | 开发工期爆炸，维护成本高 | MVP 控制在 5-7 个核心页面 |
| 5 | **忽略页面速度** | 首屏 5 秒加载 | 每多 1 秒加载时间，转化率下降 7% | 性能预算从第一天开始设定 |
| 6 | **SEO 后补** | "先上线再做 SEO" | URL 结构混乱，后期改动成本极高 | SEO 基础（URL 结构、meta、sitemap）从 V1.0 开始 |

### 技术考量

#### 架构选型

| 决策点 | 选项 A | 选项 B | 推荐 | 考量因素 |
|--------|--------|--------|------|---------|
| **渲染模式** | SSR（Next.js / Nuxt） | CSR（React SPA） | SSR | SEO 需求 + 首屏加载速度；CSR 仅适用于纯后台应用 |
| **部署方式** | Edge + CDN（Vercel/Cloudflare） | 传统服务器 | Edge + CDN | 全球访问速度、自动扩缩容、降低运维成本 |
| **CSS 方案** | Tailwind CSS | CSS Modules / Styled Components | Tailwind | 开发效率高、一致性好、bundle size 可控 |
| **CMS 方案** | Headless CMS（Strapi/Contentful） | 代码内置内容 | 视频频率而定 | 内容更新频率 > 1 次/周 → Headless CMS |

#### 性能要求

| 指标 | 目标值 | 测量方式 |
|------|--------|---------|
| LCP（Largest Contentful Paint） | < 2.5s | Lighthouse / Web Vitals |
| FID（First Input Delay） | < 100ms | Lighthouse / Web Vitals |
| CLS（Cumulative Layout Shift） | < 0.1 | Lighthouse / Web Vitals |
| TTFB（Time to First Byte） | < 800ms | WebPageTest |
| 首屏加载（3G） | < 3s | Chrome DevTools 模拟 |

#### 技术风险

| 风险 | 概率 | 影响 | 缓解方案 |
|------|------|------|---------|
| 第三方脚本拖慢加载 | 高 | 中 | 异步加载、延迟非关键脚本、定期性能审计 |
| SEO 渲染问题 | 中 | 高 | 使用 SSR/SSG，定期检查 Google Search Console |
| CDN 缓存导致旧内容 | 中 | 中 | 设置合理的缓存策略和 purge 机制 |
| 浏览器兼容性问题 | 低 | 中 | 定义最低支持浏览器版本，使用 BrowserStack 测试 |

### 行业基准

| KPI | 行业基准 | 优秀水平 | 合格水平 | 数据来源 |
|-----|---------|---------|---------|---------|
| **跳出率** | 50-65% | < 40% | < 60% | Google Analytics 行业平均 |
| **平均会话时长** | 2-3 min | > 4 min | > 2 min | Google Analytics 行业平均 |
| **注册转化率** | 2-5% | > 7% | > 2% | SaaS 行业报告 |
| **页面/会话** | 2-3 页 | > 4 页 | > 2 页 | Google Analytics |
| **移动端流量占比** | 50-70% | — | 需要对应设计投入 | StatCounter 全球数据 |
| **Lighthouse 总分** | 60-70 | > 90 | > 75 | Google Lighthouse |

**基准使用注意：**
- 以上基准适用于 B2B SaaS / 工具类网站，电商类网站的基准值有显著差异
- 冷启动阶段（< 1000 DAU）数据波动大，关注趋势而非绝对值
- 不同获客渠道（SEO vs 广告 vs 直接）的转化率差异可达 3-5 倍
