# 可量化可追踪方法论

## 1. OKR — 目标与关键结果

### 框架定义

```
Objective（目标）: 方向性、鼓舞人心、定性
  └── Key Result 1（关键结果）: 可量化、有截止日、可验证
  └── Key Result 2
  └── Key Result 3（≤ 3 个）
```

### 写法标准

| 要素 | 好的写法 | 坏的写法 |
|------|---------|---------|
| Objective | "成为细分市场用户首选工具" | "做大做强" |
| Key Result | "月活跃用户从 0 增长到 500" | "增加用户数量" |
| Key Result | "NPS 评分达到 40+" | "提升用户满意度" |
| Key Result | "SEO 自然流量占比达 30%" | "做好 SEO" |

### OKR 评分标准

| 分数 | 含义 | 说明 |
|------|------|------|
| 1.0 | 完全达成 | 设定太保守？ |
| 0.7 | 理想状态 | 有挑战但可达 |
| 0.3-0.4 | 有进展 | 需要分析原因 |
| 0 | 未启动 | 优先级问题或资源不足 |

## 2. SMART 原则 — 任务定义标准

每个任务必须满足 5 个条件：

| 字母 | 含义 | 检查方法 | 示例 |
|------|------|---------|------|
| **S** - Specific | 具体的 | 别人看了知道做什么 | "写 3 篇 SEO 文章" vs "做内容" |
| **M** - Measurable | 可衡量 | 有数字或明确产出物 | "转化率 5%" vs "提高转化" |
| **A** - Achievable | 可达成 | 资源和时间允许 | 1 周完成 vs 1 天做完 3 个月的活 |
| **R** - Relevant | 相关的 | 和当前目标/主要矛盾相关 | 直接影响收入 vs 锦上添花 |
| **T** - Time-bound | 有时限 | 有明确截止日期 | "3 月 7 日前" vs "尽快" |

### SMART 快速检查

```
任务描述: "{xxx}"

□ 具体: 能否用一句话说清楚要做什么？
□ 可衡: 完成后拿什么证明做完了？
□ 可达: 以现有资源和时间，能完成吗？
□ 相关: 这件事不做，目标还能达成吗？
□ 时限: 什么时候必须完成？

5/5 通过 → 合格任务
< 4 → 需要修改
```

## 3. 北极星指标 — 公司级聚焦

### 定义

```
北极星指标 = 一个最能反映公司是否在正确方向上的数字
```

### 选择标准

| 标准 | 说明 |
|------|------|
| 反映用户价值 | 用户真正获得价值时这个数字会涨 |
| 可操作 | 团队的行为能直接影响它 |
| 先导性 | 比收入更早反映趋势 |
| 简单易懂 | 全公司都能理解 |

### 不同阶段的北极星

| 公司阶段 | 典型北极星 | 说明 |
|---------|-----------|------|
| 冷启动（0→1） | 周活跃用户数 / 付费客户数 | PMF 验证 |
| 增长（1→10） | 月增长率 / 留存率 | 规模化验证 |
| 成熟（10→100） | LTV/CAC 比率 | 经济模型健康度 |

## 4. 漏斗指标 — 增长追踪

### 标准漏斗

```
曝光(Impressions) → 点击(Clicks) → 注册(Signups) → 激活(Activation) → 付费(Payment) → 留存(Retention)
```

### 每层必须追踪

| 漏斗层 | 指标 | 计算方式 | 健康基准 |
|--------|------|---------|---------|
| 曝光→点击 | CTR (点击率) | 点击/曝光 | SEO: 3-5%, SEM: 2-8% |
| 点击→注册 | CVR (转化率) | 注册/点击 | 落地页: 5-15% |
| 注册→激活 | 激活率 | 激活用户/注册 | 40-60% |
| 激活→付费 | 付费转化率 | 付费/激活 | SaaS: 2-5% |
| 付费→留存 | 月留存率 | 次月活跃/首月付费 | > 80% |

### 漏斗分析规则

```
哪里漏得最多 = 哪里优化 ROI 最高

例: 1000 曝光 → 50 点击(5%) → 5 注册(10%) → 0 付费(0%)
瓶颈 = 注册→付费，不是曝光不够
```

## 5. 投入产出追踪 — ROI 框架

### 基本公式

```
ROI = (收益 - 成本) / 成本 × 100%

CAC (获客成本) = 总营销支出 / 新客户数
LTV (客户终身价值) = 客单价 × 购买频次 × 客户生命周期
LTV:CAC > 3:1 → 健康
LTV:CAC < 1:1 → 亏损
```

## 6. 进度追踪 — 燃尽图法

### 概念

```
计划完成量(总任务) ──────────── 理想线（线性递减）
                   ╲
                    ╲─────── 实际线（根据完成情况波动）
                     ╲
                      ╲
                       0 ── 周期结束时全部完成
```

### 追踪方式

每周记录：

| 周 | 计划剩余 | 实际剩余 | 偏差 | 原因 |
|---|---------|---------|------|------|
| W1 | 15 | 16 | +1 | 新增紧急任务 |
| W2 | 10 | 12 | +2 | 技术卡点 |
| W3 | 5 | 8 | +3 | 需要调整范围 |
| W4 | 0 | 3 | +3 | 滚入下周期 |

### 预警规则

| 偏差 | 级别 | 行动 |
|------|------|------|
| 实际 ≤ 计划 | 正常 | 继续 |
| 实际 > 计划 × 1.2 | 黄灯 | 分析原因，调整排期 |
| 实际 > 计划 × 1.5 | 红灯 | 砍范围或加资源 |

## 7. 复盘方法 — 4L 框架

每个周期结束使用：

| L | 含义 | 问题 |
|---|------|------|
| **Liked** | 做得好的 | 哪些做法值得保留？ |
| **Learned** | 学到的 | 有什么新发现或教训？ |
| **Lacked** | 缺少的 | 哪些资源/能力/信息不足？ |
| **Longed for** | 期望的 | 下个周期最希望改善什么？ |

### 复盘输出标准

```
每次复盘必须产出：
1. ≥ 1 个保留的好做法 → 固化为 SOP
2. ≥ 1 个教训 → 更新 references
3. ≥ 1 个改善项 → 纳入下周期 P0
```
